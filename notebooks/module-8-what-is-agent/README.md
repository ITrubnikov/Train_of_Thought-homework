# Модуль 8 — Что такое агент

Заготовки к [Модулю 8: Что такое агент: что может, что не может](https://itrubnikov.github.io/Train_of_Thought/docs/modules/08-what-is-agent/).

Это первый «настоящий агент» в курсе. Не «вызвал LLM, получил текст»,
а **LLM, которая в цикле сама зовёт функции и видит результат**. Здесь
вы соберёте такого агента в 30 строк через function calling в Gemini.

## Что нужно сделать до начала

- Аккаунт Kaggle, прикреплённый `GOOGLE_API_KEY` (см. [Модуль 1.5](../module-1-5-modern-tools/)).
- Желательно — прочитать лекцию модуля 8 до раздела «Build twice».

Маленькая SQLite-база магазина + три функции (`list_tables`, `describe_table`, `execute_query`) + Gemini с automatic function calling. Видно, как модель сама раскладывает вопрос на цепочку SQL-вызовов. |

## Что вы делаете в ноутбуке

1. Создаёте SQLite-базу в памяти (товары, продавцы, заказы).
2. Пишете три Python-функции с типами и docstring'ами.
3. Передаёте функции в `client.chats.create(tools=[...])`.
4. Задаёте вопросы на естественном языке — модель сама ходит в базу
   и складывает ответ.
5. Через `chat.get_history()` видите весь цикл «вызов → ответ
   функции → следующий вызов → итог».

## ДЗ к модулю

Полное ДЗ описано в [самой лекции](https://itrubnikov.github.io/Train_of_Thought/docs/modules/08-what-is-agent/). Кратко:

**ДЗ 1 (обязательно).** Расширьте базу таблицей `inventory(product_id,
stock, updated_at)`. Добавьте четвёртый инструмент `get_stock(product_id)`.
Задайте агенту три вопроса, на которые он не сможет ответить без
вашего нового инструмента. Распечатайте `chat.get_history()` для
каждого случая.

**ДЗ 2 (опционально).** Найдите три ситуации, в которых модель ведёт
себя плохо: двусмысленный вопрос, опасное действие, бесполезный цикл.
Для каждой — история, диагноз, фикс.
