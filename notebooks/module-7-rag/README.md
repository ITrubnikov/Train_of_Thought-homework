# Модуль 7 — RAG, который работает

Заготовки к [Модулю 7: RAG, который работает](https://itrubnikov.github.io/Train_of_Thought/docs/modules/07-rag/).

RAG (Retrieval-Augmented Generation) — это «модель + база знаний».
Перед тем как ответить на вопрос, модель ищет релевантные куски в
вашей базе документов и **дополняет** ими промпт. Так LLM перестаёт
отвечать «по памяти» и начинает работать на ваших актуальных данных.

Главный кубик RAG — **эмбеддинги** (превращение текста в вектор,
по которому можно искать «по смыслу»). Этот ноутбук — про то, как
эмбеддинги ощущаются на руках, до того как мы будем строить полный
RAG-пайплайн.

## Что вы делаете в ноутбуке

1. Берёте 6—8 вариантов одной фразы (опечатки, перефразы, сокращения,
   плюс одну несвязанную для контраста).
2. Превращаете каждую в вектор через `embed_content`.
3. Считаете cosine similarity матрицу (всё ко всем).
4. Рисуете heatmap — видите, что похожие по смыслу фразы дают похожие
   векторы.

Это та же самая операция, которую RAG делает миллион раз: «найди мне
документы в базе, которые ближе всего к этому запросу».
